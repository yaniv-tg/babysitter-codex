# Issue Tracking Skill

Bead-based issue tracking adapted from [Gas Town](https://github.com/steveyegge/gastown) by Steve Yegge.

## Purpose

Track atomic work units (beads) through their lifecycle with persistent attribution.

## Bead States

Created -> Assigned -> In Progress -> Review -> Done (or Destroyed for wisps)

## Integration

- **Input from:** `work-decomposition` and `convoy-management`
- **Output to:** Progress tracking, attribution reports
- **Process file:** Used across all gastown process files
