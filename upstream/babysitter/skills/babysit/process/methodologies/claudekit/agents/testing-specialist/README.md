# Testing Specialist Agent

Assesses test coverage and quality across changed files.

## Focus Areas

1. **Coverage**: Changed files have tests
2. **Quality**: Meaningful assertions
3. **Edge Cases**: Boundaries, empty inputs, concurrency
4. **Isolation**: No shared mutable state
5. **Flakiness**: Timing, network, randomness
6. **Integration**: API boundaries, cross-module
7. **Error Paths**: Failures, exceptions, timeouts

## Invocation

Used by process: `methodologies/claudekit/claudekit-code-review`
