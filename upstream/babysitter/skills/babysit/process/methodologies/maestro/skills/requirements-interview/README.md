# Requirements Interview Skill

Interactive Product Manager interview for requirements elicitation. Adapts questioning depth to user expertise level (beginner, intermediate, expert).

## Inputs
- `goal` - Project goal or description
- `userExpertise` - User expertise level
- `projectContext` - Existing project context

## Outputs
- Requirements specification
- User personas
- Feature list with priorities
- Constraints and acceptance criteria
